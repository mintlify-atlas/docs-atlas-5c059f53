description: "Run your first protein inference with E1 in minutes."

<Tip>
  Use **E1-150m** for quick experimentation and prototyping. Switch to **E1-600m** for production workloads where representation quality matters most. E1-300m is a good all-around choice for balancing speed and accuracy.
</Tip>

<Steps>
  <Step title="Install E1">
    Clone the repository and install the package with pip:

    ```bash
    git clone https://github.com/Profluent-AI/E1.git
    cd E1 && pip3 install -e .
    ```

    See [Installation](/installation) for optional dependencies (e.g., `flash-attn` for GPU acceleration) and compute requirements.
  </Step>

  <Step title="Load the model">
    Import `E1ForMaskedLM` and load a pretrained checkpoint. Weights are downloaded automatically from HuggingFace on first use.

    ```python
    import torch
    from E1.batch_preparer import E1BatchPreparer
    from E1.modeling import E1ForMaskedLM

    model = E1ForMaskedLM.from_pretrained("Profluent-Bio/E1-300m").to("cuda:0")
    model.eval()
    ```
  </Step>

  <Step title="Prepare a batch and run inference">
    Use `E1BatchPreparer` to tokenize and pad your sequences, then run a forward pass under `torch.autocast` for BF16 precision:

    ```python
    sequences = ["AAAAA?C", "MFCATEEKL,MCCASDF,MFCC?SEF"]
    batch_preparer = E1BatchPreparer()
    batch = batch_preparer.get_batch_kwargs(sequences, device="cuda:0")

    with torch.autocast("cuda", dtype=torch.bfloat16, enabled=True):
        outputs = model(
            input_ids=batch["input_ids"],
            within_seq_position_ids=batch["within_seq_position_ids"],
            global_position_ids=batch["global_position_ids"],
            sequence_ids=batch["sequence_ids"],
            past_key_values=None,
            use_cache=False,
            output_attentions=False,
            output_hidden_states=False,
        )

    logits: torch.Tensor = outputs.logits  # (B, L, V)
    embeddings: torch.Tensor = outputs.embeddings  # (B, L, E)
    ```

    The second sequence, `"MFCATEEKL,MCCASDF,MFCC?SEF"`, demonstrates **retrieval-augmented mode**: homologs (`MFCATEEKL` and `MCCASDF`) are comma-separated prefixes, and the query sequence (`MFCC?SEF`) is the final entry. `?` is the mask token.

    Output shapes:
    - `logits` — `(B, L, V)`: batch × padded sequence length × vocabulary size
    - `embeddings` — `(B, L, E)`: batch × padded sequence length × hidden size
  </Step>

  <Step title="Extract residue-level outputs with selectors">
    The batch tensors include special boundary tokens (`<bos>`, `<eos>`, `1`, `2`, `<pad>`) and, in retrieval-augmented inputs, context sequences that precede the query. Use boolean selectors to isolate the residues that belong to the query sequence only:

    ```python
    # last_sequence_selector: True for tokens that are part of the last sequence
    last_sequence_selector = batch["sequence_ids"] == batch["sequence_ids"].max(dim=1)[0][:, None]
    # residue_selector: True for tokens that are part of the input sequence (not boundary tokens)
    residue_selector = ~(batch_preparer.get_boundary_token_mask(batch["input_ids"]))
    # last_sequence_residue_selector: True for residues in the last sequence (excluding boundary tokens)
    last_sequence_residue_selector = last_sequence_selector & residue_selector

    # Get embeddings for the query sequences only
    last_sequence_embeddings = [embeddings[i, last_sequence_residue_selector[i]] for i in range(embeddings.shape[0])]
    ```

    This yields embeddings for `["AAAAA?C", "MFCC?SEF"]` — the query portions only — discarding boundary tokens and the homolog context in the second input. You can apply the same selector pattern to `logits`.

    Selector summary:

    | Selector | Selects |
    |----------|---------|
    | `last_sequence_selector` | All tokens belonging to the last (query) sequence, including its boundary tokens |
    | `residue_selector` | All tokens that are amino acid residues (excludes `<bos>`, `<eos>`, `1`, `2`, `<pad>`) |
    | `last_sequence_residue_selector` | Amino acid residues in the query sequence only |
  </Step>
</Steps>
